## 比赛整体思路与技术路线说明

***by zijeff***

### 任务理解

本项目的核心目标是：基于 20 秒游戏片段中各玩家的状态信息，预测主玩家在当前时刻的行为决策。所以本质上这是一个分类任务，仅在初赛要求下是这样的。结合数据格式和比赛要求，我们将对任务做如下拆解：

- **动作行为预测**：对于初赛的动作，我并没有构造新的字段，而是在原有的动作基础上，即“**开火，放技能，搜，丢雷，救援**”这五类动作开展预测，也就是将其认定为一个**五分类**的任务。
- **意图决策映射**：对于主玩家的**意图决策**，我采用的是从动作行为映射回意图决策上，这样可以保留更丰富的行为信息，同时满足最终答题卡格式要求。

------

### 总体技术路线

整体方案采用如下的一个流程：

**日志解析 -> 静态快照构建 -> 数据清洗 -> 特征增强 -> 类别不平衡处理 -> CatBoost 建模 -> 测试集推理 -> 提交结果生成**

完整路线解析如下：

1. 从原始 `.txt` 日志中提取主玩家在决策时刻的静态快照，因为原始的文件是一个时间序列，直接对其进行分析会比较困难。然后，从我个人游戏的游玩经历而言，玩家在当前时刻的决策往往只依赖于这一刻的某些信息，所以我们可以跟踪这一段时间序列中一些信息的变化，比如敌人的距离，队友的数量以及自身的状态等，将其作为初始的特征，详情可见 ```data_extract.py```

   综上，提取静态快照分析既有利于**模型的加速训练**，适合以较低的算力获得较好的模型表现，也比较**符合真人玩家**在做决策时的心理状态。

2. 将每个样本整理为结构化的表格数据，利于使用 ```pandas``` 对其进行后续的数据处理。将所有的数据集数据读入后，我对其进行了初步的数据查看，详情可见 ```data_analysis.py``` 脚本中的处理

3. 对数据进行了初步的查看后，我们进一步对表格数据做缺失值分析、字段类型识别、异常字段筛查，删除高风险泄漏字段与无效字段。这样，我们便对数据进行了一个简单的初步处理，具体代码可见 ```data_process.py```

4. 由上一步得到的初步数据结果，我使用 **CatBoost** 进行初步的多分类训练，得到结果并不算很好，所以我进一步进行了优化。

5. 具体采取了这些优化方式：

   - 构造几何关系特征、局部战场压力特征、状态组合特征
   - 通过 5 折交叉验证评估稳定性
   - 针对类别不平衡进行手动权重调整
   - 在权重附近进行小范围的**启发式搜索**，尝试寻找最优解

6. 训练最终模型并对测试集进行推理，最终训练的模型在数据集划分出的验证集上取得了较为不错的成绩，最终几个主要的评估指标如下：

   |   指标   |    数值    |
   | :------: | :--------: |
   | Accuracy | **0.8576** |
   | Macro-F1 | **0.7663** |
   | Macro-F1 | **0.8534** |

> [!note]
>
> ```preliminary_training.py``` 是在没有构造新特征时的训练脚本。
>
> ```data_process_geo.py``` 是构造了几何关系特征、局部战场压力特征、状态组合特征这些新的特征。
>
> ```preliminary_training_geo.py``` 是基于新的特征后并同时附加了五折交叉验证的训练脚本。
>
> ```weight_test.py``` 是对附加权重后模型预测效果是否得到改善的测试脚本。
>
> ```weight_search.py``` 是发现了添加附件权重后，模型的预测效果得到了较好的改善，所以在当前权重附近进行小范围的搜索尝试找到更优权重。由于时间限制，我只做了一个小范围的搜索，这可能也是当前工作的一个局限。
>
> ```final.py``` 是最终的模型训练脚本，并最后输出了答案文件。

### 原始日志到静态快照的构建

#### 原始数据特点

经过我对前几个的数据集文件解压后观察，可以发现：训练数据原始格式为大量游戏日志 `.txt` 文件，包含以下信息，

- 游戏开始事件
- 玩家基础信息
- 技能生效 / 技能结束
- 决策事件

其中，日志是按时间顺序记录的，因此我可以通过顺序扫描的方式维护每个玩家在当前时刻的最新状态，最后得到静态快照。

#### 玩家状态维护

因为最后我们的任务是对某个时刻的主玩家的决策意图做出预测，所以我们可以先对每个玩家维护如下状态：

- 玩家 ID、队伍、角色
- 空间位置 $(x,y,z)$
- 武器偏航角、俯仰角
- 速度向量与速度大小
- 相机坐标
- 视野范围
- 开镜状态
- Buff 集合

日志解析时：

- **游戏开始**用于初始化玩家身份信息
- **玩家基础信息**用于刷新空间与视角状态
- **技能生效 / 技能结束**用于动态维护 Buff 集合

#### 决策时刻静态快照冻结

处理训练集时，当扫描到**（决策）...**日志时，立即冻结当前所有玩家状态，提取一条样本。提取时以该决策玩家为主玩家，提取并构造如下信息：

- 主玩家自身状态
- 敌方与队友的整体统计信息
- 最近敌人详细信息
- 最近队友详细信息

进而形成一条完整的结构化样本，对**测试集数据**采取同样的方式。

> [!important]
>
> 如果要全量提取所有敌人和队友信息，这会造成数据处理很麻烦，同时产生的样本特征会非常之多，使模型训练时间成倍增长，训练效果可能并不太好。为了解决这个问题，我还是从自身的角度出发，考虑了自己作为一个玩家在进行游戏时的**心里决策过程**：
>
> - 玩家可能并不会仔细考虑全局的信息，也就是所有的玩家和敌人，这对于快节奏的战斗并不现实。更多的时候，玩家心里只有对当前形势的一个大致判断，所以我采用了一个敌方和队友的整体统计作为特征之一。
> - 但是，一些 “**高玩**” （也就是游戏技术很好，枪法意识都比较好的的玩家）会有一定的全局意识，而战斗时打破僵局的最好方法是创造**一打一**的局面，使对方出现人数上的减员。因此，我们需要记录最近的敌人详细信息和最近队友信息以供模型参考。
> - 除此之外，玩家在战斗时更多考虑的是**相对距离**而非是**绝对距离**，这是因为相对距离比较容易通过感觉直观感受，所以我也加入了相对距离作为初始特征之一。
>
> 综上所述，我觉得对于玩家在游玩时候的**心理状态分析**可能是我当前工作的一个**创新点**。

### 类别不平衡问题与权重搜索

#### 初步问题

模型在基线阶段已经具备较强预测能力，但存在明显类别不平衡问题：

- **开火**样本占比最高
- **丢雷**样本较少，且最难预测
- 直接不加权时，模型几乎放弃预测**丢雷**
- 使用自动平衡权重时，又会产生大量**丢雷**误报

#### 三组权重实验

为了确定具体的权重方案，我对比了三种权重方案：

- **平衡型**权重分配
- **无权重**分配
- **手动权重**分配

运行后得到实验结论：**平衡型**权重分配的丢雷 recall 高，但 precision 很差，整体指标被拖累；**无权重**分配的Accuracy 很高，但丢雷 recall 极低，基本被忽略；**手动权重**分配整体最均衡，Macro-F1 最高。

#### 最终确定的权重

最终采用手动权重：

```python
[2.0, 1.0, 1.0, 1.2, 1.5] # 对应标签顺序为：0->丢雷 1->开火 2->搜 3->放技能 4->救援
```

该方案的 5 折结果可见**总体路线**中的表述，相比其他方案，该方案兼顾了整体性能与少数类表现，因此作为最终正式训练方案。

### 整体方案总结

本方案的整体思路，是先将原始的复杂时序日志转化为决策时刻的静态快照，再围绕主玩家、敌方玩家、队友玩家以及局部战场环境构造结构化表格特征，最终通过适合表格数据建模的 CatBoost 模型完成动作行为预测。在这一过程中，我们重点控制了高风险泄漏字段的使用，对包含明显标签信息或强记忆风险的字段进行了剔除，同时通过几何关系特征显式刻画主玩家与最近敌人、最近队友之间的空间关系，从而提升模型对于“开火”“放技能”“丢雷”“搜”“救援”等行为模式的判别能力。针对类别分布不均衡的问题，方案中进一步结合 5 折交叉验证和手动类别权重搜索，对不同类别之间的预测平衡进行了调整，使模型不仅关注整体准确率，也尽量兼顾少数类的识别效果。

从当前实验结果来看，这一方案具备较强的工程实用价值。首先，它将原本难以直接建模的日志数据转换为了统一、清晰、可解释的表格样本，整个特征构造过程具有较强的可追踪性和可复现性。其次，方案**没有依赖过于复杂的深度时序模型**，而是优先使用更稳健、训练效率更高、对类别与数值混合特征**更友好的 CatBoost**，在比赛时间有限的条件下实现了较快的迭代速度。再次，通过多轮数据检查、特征增强和权重实验，模型在保证整体性能稳定的同时，也较好地提升了对部分类别的区分能力，使最终方案兼顾了可解释性、工程可落地性和提交结果的稳定性。

不过，当前方案仍然存在一定不足。首先，本方案本质上仍然是基于“静态快照”的建模方法，虽然能够较好地利用决策时刻附近的状态信息，但对于玩家行为背后的连续时序演化过程刻画仍然不够充分。其次，尽管我们已经加入了几何关系特征和局部环境统计特征，但部分复杂行为，尤其是“丢雷”这类边界较模糊、受战场空间关系和策略因素共同影响的类别，识别效果仍然不够理想，说明现有特征对这类行为的表达还不够充分。再次，当前的类别不平衡处理主要依赖手动权重搜索，这种方式虽然在比赛时间约束下较为高效，但仍带有一定经验性，缺乏更系统的自动调优过程。最后，测试集快照提取采用的是“整段日志结束后冻结一帧”的方式，这一策略在工程上是合理且可执行的，但与训练集中“决策发生时刻冻结快照”的逻辑相比，仍然存在一定分布差异，这也可能对模型泛化带来影响。

总体而言，当前方案在有限时间内实现了从原始日志解析、结构化建模、特征增强到最终推理提交的完整闭环，能够满足比赛对可运行性和结果稳定性的基本要求。但从更高水平的建模角度看，这一方案仍然属于以工程效率和落地稳定性为优先的表格建模方法，后续如果有更充足的时间，还可以进一步从时序建模、行为上下文建模、类别不平衡优化以及特征自动搜索等方向继续提升模型性能。
