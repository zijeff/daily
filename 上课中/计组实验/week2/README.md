# 单周期 CPU Verilog 实验说明

## 1. 项目简介

本项目是一个基于 Verilog 实现的单周期 CPU 教学实验工程，整体结构清晰，适合用于学习计算机组成原理中的以下内容：

- 单周期 CPU 的基本数据通路
- 指令译码与控制信号生成
- 寄存器堆、ALU、程序计数器、指令存储器、数据存储器之间的连接关系
- `lw`、`sw`、`beq`、`jal` 等典型指令的执行流程
- 使用 `iverilog`、`vvp` 和 `gtkwave` 进行仿真与波形观察

该工程不是完整的 RV32I 处理器实现，而是一个面向教学和实验的简化版本，已经支持一组基础指令，足以展示单周期 CPU 的核心工作机制。

## 2. 项目结构

当前目录中的主要文件如下：

| 文件名 | 作用 |
| --- | --- |
| `sccomp_tb.v` | 仿真测试平台，负责加载测试程序、产生时钟和复位、导出波形 |
| `sccomp.v` | 系统顶层模块，连接 CPU、指令存储器和数据存储器 |
| `SCCPU.v` | 单周期 CPU 核心数据通路 |
| `ctrl.v` | 控制器，根据指令字段生成控制信号 |
| `ctrl_encode_def.v` | 控制信号与 ALU/NPC/EXT 编码宏定义 |
| `PC.v` | 程序计数器寄存器 |
| `NPC.v` | 下一条 PC 计算模块 |
| `EXT.v` | 立即数扩展模块 |
| `RF.v` | 通用寄存器堆 |
| `alu.v` | 算术逻辑单元 |
| `im.v` | 指令存储器 |
| `dm.v` | 数据存储器 |
| `rv32_sc_sim.dat` | 仿真使用的机器码程序 |
| `Makefile` | 编译、运行仿真、查看波形的自动化脚本 |

## 3. 顶层结构

整个系统可以概括为如下连接关系：

```text
sccomp_tb
    |
    v
  sccomp
   |- SCCPU
   |- im
   |- dm
```

其中：

- `sccomp_tb` 是仿真入口
- `sccomp` 是系统装配层
- `SCCPU` 是真正的 CPU 核心
- `im` 提供指令
- `dm` 提供数据读写

## 4. CPU 核心模块说明

### 4.1 SCCPU.v

`SCCPU.v` 是整个工程最核心的文件，主要包含以下功能：

- 对输入指令进行字段拆分
- 调用控制器生成控制信号
- 连接 `PC`、`NPC`、`EXT`、`RF`、`ALU`
- 生成访存地址和写数据
- 选择最终写回寄存器的数据来源

可以把它理解为“CPU 的主数据通路”。

### 4.2 ctrl.v

`ctrl.v` 是控制器，只做组合逻辑译码，不保存状态。它的作用是根据：

- `Op`
- `Funct3`
- `Funct7`
- `Zero`

生成如下控制信号：

- `RegWrite`
- `MemWrite`
- `EXTOp`
- `ALUOp`
- `NPCOp`
- `ALUSrc`
- `WDSel`

也就是说，`ctrl.v` 决定“这条指令应该怎么执行”。

### 4.3 EXT.v

`EXT.v` 负责把不同格式的立即数扩展成统一的 32 位数，支持：

- I 型
- S 型
- B 型
- U 型
- J 型

该模块为 `addi`、`lw`、`sw`、`beq`、`lui`、`jal` 等指令提供立即数支持。

### 4.4 alu.v

`alu.v` 负责执行算术逻辑运算，并输出结果 `C` 和零标志 `Zero`。当前工程支持的主要运算包括：

- `add`
- `sub`
- `xor`
- `or`
- `and`
- `sll`
- `srl`
- `sra`
- `lui`

其中 `Zero` 信号会被控制器用于 `beq` 指令的跳转判断。

### 4.5 RF.v

`RF.v` 是 32 个通用寄存器组成的寄存器堆，具有以下特点：

- 两个读端口
- 一个写端口
- 时钟上升沿写入
- 组合逻辑读出
- `x0` 恒为 0，不允许写入

此外，该模块还提供调试接口，可以根据 `reg_sel` 输出指定寄存器的值。

### 4.6 PC.v 与 NPC.v

这两个模块共同完成取指地址控制：

- `PC.v` 保存当前 PC
- `NPC.v` 负责计算下一条 PC

`NPC.v` 支持三种主要路径：

- `PC + 4`
- `PC + branch_offset`
- `PC + jump_offset`

### 4.7 im.v 与 dm.v

这两个模块分别对应存储系统中的：

- 指令存储器 `im`
- 数据存储器 `dm`

其中：

- `im.v` 只负责按地址读指令
- `dm.v` 支持 `lw` / `sw`

`dm.v` 采用：

- 时钟上升沿写入
- 组合逻辑读出

这是一种典型的教学型单周期实现方式。

## 6. 测试程序说明

`rv32_sc_sim.dat` 中包含了一段用于仿真的测试程序，主要用于验证以下功能：

- 立即数写寄存器
- R 型算术逻辑运算
- 左移、逻辑右移、算术右移
- 数据存储与加载
- 条件分支
- 无条件跳转

该程序最后包含回跳逻辑，因此如果 testbench 不主动调用 `$finish`，仿真会持续循环执行。

## 7. 仿真环境

建议安装以下工具：

- `iverilog`
- `vvp`
- `gtkwave`

在 Ubuntu 或类 Debian 系统中可以参考：

```bash
sudo apt install iverilog gtkwave
```

其中：

- `iverilog` 用于编译 Verilog
- `vvp` 用于运行仿真
- `gtkwave` 用于查看波形

## 8. Makefile 使用方法

本目录已经提供 `Makefile`，可直接使用。

### 8.1 编译

```bash
make compile
```

### 8.2 运行仿真

```bash
make run
```

默认仿真时间为 `200ns`。如果需要指定仿真时间，可以使用：

```bash
make run SIM_TIME=300
```

### 8.3 生成波形并打开

```bash
make wave
```

该命令会：

1. 编译工程
2. 运行仿真
3. 生成 `sccomp_tb.vcd`
4. 调用 `gtkwave` 打开波形文件

### 8.4 清理生成文件

```bash
make clean
```

## 11. 关键理解要点

在理解该项目时，建议特别关注以下几点：

- 单周期 CPU 在一个时钟周期内完成取指、译码、执行、访存和写回
- 控制器只负责发控制信号，不负责具体运算
- ALU 既负责算术逻辑运算，也负责分支比较和访存地址计算
- `lw` 和 `sw` 是理解数据通路最好的入口
- `beq` 和 `jal` 是理解 PC 更新路径最好的入口

## 12. 已知说明

- `rv32_sc_sim.dat` 文件中的指令数量少于指令存储器总容量，因此仿真时 `readmemh` 可能给出容量不足警告，这通常不影响本实验运行
- 当前 testbench 已支持导出 `VCD` 波形文件，并在设定时间后自动结束仿真
- 若运行 `make wave` 时图形界面无法启动，请检查本机图形环境与 `DISPLAY` 配置
