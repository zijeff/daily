# 单周期 CPU 实现理论说明

## 1. 实验背景

单周期 CPU 是计算机组成原理课程中最经典的处理器模型之一。它的核心思想是：**每条指令都在一个时钟周期内完成全部执行过程**。这一过程通常包括取指、译码、执行、访存、写回五个阶段，但在单周期结构中，这些阶段不是跨多个周期展开，而是被压缩在同一个时钟周期内部完成。

这种结构的意义不在于追求高性能，而在于帮助学习者清晰理解：

- 指令如何被取出并解释
- 数据通路中的各功能部件如何协同工作
- 控制信号如何驱动不同类型指令执行
- 程序计数器、寄存器堆、ALU、存储器之间的逻辑关系

因此，单周期 CPU 更适合作为“理解处理器工作原理”的教学模型，而不是实际高性能处理器的最终形态。

## 2. 单周期 CPU 的基本特点

### 2.1 一个周期完成一条指令

在单周期处理器中，不论是简单的算术指令，还是较复杂的访存或跳转指令，都必须在同一个时钟周期内完成。这意味着时钟周期长度必须足够覆盖最慢指令的全部路径。

### 2.2 控制逻辑直观

单周期 CPU 通常采用组合逻辑控制器。控制器根据当前指令的操作码和功能码，直接生成所有控制信号，因此结构直观、分析方便。

### 2.3 实现简单但性能有限

单周期结构的优势是简单、清晰、便于验证；缺点是所有指令共用同一个较长时钟周期，导致时钟频率受限，整体性能较低。特别是加载、存储、分支、跳转等指令，会拉长关键路径。

## 3. 单周期 CPU 的整体数据通路

一个教学型单周期 CPU 一般围绕以下几个核心部件构成：

- 程序计数器 PC
- 指令存储器 IM
- 控制器 CTRL
- 寄存器堆 RF
- 立即数扩展器 EXT
- 算术逻辑单元 ALU
- 数据存储器 DM
- 下一条地址生成单元 NPC

其基本工作流程如下：

1. PC 给出当前指令地址。
2. 指令存储器根据地址输出机器指令。
3. 控制器对指令进行译码，产生控制信号。
4. 寄存器堆读出源操作数。
5. 立即数扩展器生成所需的 32 位立即数。
6. ALU 根据控制信号完成算术、逻辑或地址计算。
7. 如果是访存指令，则访问数据存储器。
8. 根据写回选择信号，将结果写回寄存器堆。
9. NPC 计算下一条 PC，供下一个时钟周期使用。

从教学角度看，这条数据通路体现了“指令驱动硬件动作”的本质。

## 4. 主要功能部件的理论作用

### 4.1 程序计数器 PC

PC 用于保存当前正在执行的指令地址。正常顺序执行时，PC 每次加 4，因为一条 32 位指令占 4 字节。遇到分支或跳转时，PC 不再简单加 4，而是跳转到由立即数和当前地址共同计算出的目标地址。

### 4.2 指令存储器

指令存储器负责按照 PC 提供的地址取出指令。教学实验中通常将指令存储器和数据存储器分离，这样可以避免单端口存储器在同一周期内同时取指和读写数据所产生的结构冲突。这种做法可视为一种简化的哈佛结构思想。

### 4.3 控制器

控制器是单周期 CPU 的“指挥中心”。它根据指令的编码字段决定：

- 是否写寄存器
- 是否写数据存储器
- ALU 执行哪种运算
- ALU 第二个操作数来自寄存器还是立即数
- 立即数采用哪种格式扩展
- 写回阶段应选择 ALU 结果、存储器读数还是返回地址
- 下一条 PC 应取顺序地址、分支目标地址还是跳转目标地址

控制器本质上完成了“从指令语义到硬件控制”的映射。

### 4.4 寄存器堆

寄存器堆通常包含 32 个通用寄存器，支持：

- 两个读端口
- 一个写端口

这样一条二元运算指令可以在一个周期内同时读取两个源寄存器，并在周期末把结果写回目标寄存器。RISC-V 中 `x0` 寄存器恒为 0，这一特性有助于简化硬件和指令使用。

### 4.5 立即数扩展器

RISC-V 中不同类型指令的立即数字段分布不同，因此需要专门的扩展器将它们恢复为统一的 32 位形式。教学实验中通常会涉及以下几类立即数：

- I 型：用于立即数运算和加载
- S 型：用于存储
- B 型：用于条件分支
- U 型：用于高位立即数构造
- J 型：用于无条件跳转

扩展器不仅要完成符号扩展，还要在分支和跳转场景下处理地址对齐需求。

### 4.6 ALU

ALU 负责执行 CPU 中最核心的算术逻辑运算，例如：

- 加法
- 减法
- 与、或、异或
- 左移、右移、算术右移
- 高位立即数装载相关运算

此外，ALU 还承担地址计算和比较功能。例如：

- `lw`、`sw` 通过 ALU 计算有效地址
- `beq` 通过减法或比较结果判断是否满足分支条件

因此，ALU 不只是“算数器”，也是地址生成和条件判断的重要单元。

### 4.7 数据存储器

数据存储器主要服务于加载和存储指令：

- 加载指令从存储器读取数据，再写回寄存器
- 存储指令把寄存器中的数据写入存储器

在简化教学模型中，常以字为基本访问单位，便于突出主线数据通路，而不把重点放在字节使能、非对齐访问等更复杂的问题上。

### 4.8 下一条地址生成单元 NPC

NPC 负责计算下一条指令地址。它通常至少需要支持三类来源：

- `PC + 4`：顺序执行
- `PC + branch_offset`：条件分支
- `PC + jump_offset`：无条件跳转

NPC 的存在使“当前指令执行结果”能够直接影响“下一条指令从哪里取”。

## 5. 指令类型与执行思想

### 5.1 R 型指令

R 型指令的两个源操作数都来自寄存器，结果写回目标寄存器。其理论流程是：

1. 读取两个源寄存器。
2. 控制器指定 ALU 运算类型。
3. ALU 输出结果。
4. 结果写回目标寄存器。

这类指令最能体现寄存器堆与 ALU 的基本协作关系。

### 5.2 I 型立即数运算

I 型立即数运算的一个操作数来自寄存器，另一个来自扩展后的立即数。其核心意义是减少对临时寄存器的依赖，使常数参与运算更加高效。

典型流程为：

1. 读取一个源寄存器。
2. 扩展立即数。
3. ALU 完成运算。
4. 将结果写回寄存器。

### 5.3 加载指令

加载指令的关键是“地址生成 + 存储器读取 + 寄存器写回”。理论流程为：

1. 从寄存器读出基址。
2. 将基址与立即数相加，得到有效地址。
3. 访问数据存储器并读取数据。
4. 将读取结果写回目标寄存器。

加载指令的数据通路比普通算术指令更长，因此经常成为单周期 CPU 时钟周期设计的重要参考。

### 5.4 存储指令

存储指令不写回寄存器，而是把寄存器中的数据写入数据存储器。其理论流程为：

1. 读取基址寄存器。
2. 读取待写入数据的寄存器。
3. 基址与立即数相加生成有效地址。
4. 向数据存储器写入数据。

这说明并不是每条指令都需要写回阶段，但写回路径仍然是整体数据通路设计的一部分。

### 5.5 条件分支指令

条件分支的核心在于“比较结果决定下一条 PC”。理论流程为：

1. 读取两个待比较寄存器。
2. 通过 ALU 比较它们是否满足条件。
3. 如果条件成立，则 NPC 选择分支目标地址。
4. 如果条件不成立，则 NPC 选择 `PC + 4`。

单周期 CPU 中，比较、条件判断和 PC 选择都必须在同一个周期内完成，这也是分支路径较长的原因之一。

### 5.6 无条件跳转指令

无条件跳转会直接修改控制流，同时常常需要保存返回地址。其理论流程为：

1. 根据跳转立即数计算目标地址。
2. 将下一条顺序地址作为返回地址写回目标寄存器。
3. NPC 选择跳转目标地址作为新的 PC。

这类指令展示了控制流改变与写回通路同时发生的特点。

## 6. 关键控制信号的意义

虽然不同实验命名方式不完全一致，但单周期 CPU 通常都围绕以下几类控制信号工作：

- `RegWrite`：是否允许写回寄存器
- `MemWrite`：是否允许写数据存储器
- `ALUSrc`：ALU 的第二操作数来自寄存器还是立即数
- `ALUOp`：指定 ALU 的具体运算类型
- `EXTOp`：指定立即数扩展格式
- `WDSel`：指定寄存器写回数据来源
- `NPCOp`：指定下一条 PC 的选择方式

这些控制信号共同回答了三个根本问题：

- 数据从哪里来
- 中间经过什么运算
- 最终流向哪里去

## 7. 单周期 CPU 的时序理解

单周期 CPU 的关键时序特征是：

- 时钟边沿到来后，PC 更新
- 在整个周期内部，指令取出、译码、运算、访存、写回数据准备等过程通过组合逻辑传播完成
- 到下一个有效写入时刻，寄存器堆或数据存储器完成状态更新

从这个角度看，单周期 CPU 是一种“状态寄存器较少、组合逻辑较长”的处理器结构。它将复杂性集中在一个周期内部，而不是分散到多个流水阶段。

## 8. 单周期结构中的关键设计点

### 8.1 关键路径

单周期 CPU 的时钟周期必须满足最长指令路径的延迟要求。通常这条路径可能是：

`PC -> 指令存储器 -> 控制器/寄存器堆 -> ALU -> 数据存储器 -> 写回选择 -> 寄存器堆`

路径越长，最大时钟频率越低。

### 8.2 存储器分离

为了让取指和数据访问能够在同一周期内并行完成，教学实验常采用指令存储器与数据存储器分离的结构。这样做简化了实现，也更利于观察单周期 CPU 的主干逻辑。

### 8.3 对齐与偏移

由于指令按字对齐，分支和跳转的偏移量通常隐含低位对齐信息，因此目标地址计算并不是简单拼接字段，而是需要遵循指令格式规则恢复真实偏移量。

### 8.4 零寄存器约束

RISC-V 中 `x0` 恒等于 0，这一规则要求寄存器堆在硬件上屏蔽对 `x0` 的写入。它虽然是一个细节，但对理解体系结构约束与硬件实现之间的关系很有帮助。

## 9. 单周期 CPU 的优缺点

### 优点

- 结构清晰，便于教学
- 数据通路完整，适合理解指令执行全过程
- 控制逻辑相对直接，易于调试和仿真

### 缺点

- 时钟周期由最慢指令决定，效率不高
- 硬件资源在不同指令之间利用率不均衡
- 难以支持更复杂的高性能需求

因此，单周期 CPU 往往是后续学习多周期 CPU、流水线 CPU、冒险处理和缓存系统的基础。

## 10. 本实验应掌握的核心认识

围绕本次单周期 CPU 实验，理论上应重点掌握以下内容：

- 理解一条指令如何在一个周期内完成完整执行
- 理解控制器如何根据指令类型生成控制信号
- 理解寄存器堆、ALU、立即数扩展器、存储器之间的数据流动
- 理解顺序执行、分支和跳转对 PC 更新方式的影响
- 理解单周期结构“实现简单但周期长”的本质取舍

如果这些内容能够真正串联起来，就已经建立了对处理器基本工作机制的完整认识。后续无论学习流水线、冒险处理、异常中断还是缓存系统，都会以这里的数据通路和控制思想作为基础。
